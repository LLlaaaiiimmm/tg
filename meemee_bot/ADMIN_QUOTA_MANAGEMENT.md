# Управление генерациями в админ-панели

## Описание функционала

В админ-панели MeeMee реализован полный функционал для управления генерациями пользователей по их ID.

## Возможности

### 1. Поиск пользователя по ID

**Как использовать:**
1. Запустите админ-бот командой `/start`
2. Нажмите кнопку `👥 Пользователи`
3. Отправьте ID пользователя (число)

**Что покажет:**
- Имя и username пользователя
- Количество бесплатных генераций
- Количество платных генераций
- Всего доступно генераций
- Статистика успешных/неудачных генераций
- Сумма потраченных средств
- Дата регистрации

### 2. Добавление генераций

**Как добавить:**
1. Найдите пользователя (см. выше)
2. Нажмите `➕ Добавить генерации`
3. Выберите количество:
   - Быстрые кнопки: +1, +5, +10, +50, +100
   - Или нажмите `✏️ Ввести вручную` для произвольного количества

**Что происходит:**
- Генерации добавляются к бесплатной квоте пользователя
- Пользователь получает уведомление о начислении
- Показывается подтверждение с подробностями операции

### 3. Удаление генераций

**Как удалить:**
1. Найдите пользователя (см. выше)
2. Нажмите `➖ Убрать генерации`
3. Выберите количество:
   - Быстрые кнопки: -1, -5, -10
   - Кнопка `Обнулить все` - удаляет все бесплатные генерации
   - Или нажмите `✏️ Ввести вручную` для произвольного количества

**Что происходит:**
- Генерации вычитаются из бесплатной квоты пользователя
- Баланс не может стать отрицательным (минимум 0)
- Пользователь получает уведомление о списании
- Показывается подтверждение с подробностями операции

## Технические детали

### Файлы системы

**Основной файл:** `/app/meemee_bot/src/bot_start_admin.js`
- Обработчики callback-кнопок для управления квотами
- Логика поиска и отображения пользователей
- Уведомления пользователей через основной бот

**Сервис пользователей:** `/app/meemee_bot/src/services/User.service.js`
- `getUser(userId)` - получение данных пользователя
- `addFreeQuota(userId, amount)` - добавление бесплатных генераций
- `removeFreeQuota(userId, amount)` - удаление бесплатных генераций

### Обработчики событий

```javascript
// Кнопка добавления
bot.action(/add_quota_(\d+)/, ...)

// Подтверждение добавления
bot.action(/add_quota_confirm_(\d+)_(\d+)/, ...)

// Ручной ввод для добавления
bot.action(/add_quota_custom_(\d+)/, ...)

// Кнопка удаления
bot.action(/remove_quota_(\d+)/, ...)

// Подтверждение удаления
bot.action(/remove_quota_confirm_(\d+)_(\d+)/, ...)

// Ручной ввод для удаления
bot.action(/remove_quota_custom_(\d+)/, ...)

// Просмотр пользователя
bot.action(/show_user_(\d+)/, ...)
```

## Уведомления пользователей

При добавлении генераций пользователь получает:
```
🎁 Вам начислено X бесплатных генераций!

💎 Ваш новый баланс: Y генераций
```

При удалении генераций пользователь получает:
```
⚠️ С вашего баланса списано X генераций администратором.

💎 Ваш новый баланс: Y генераций
```

## Безопасность

- Доступ к админ-панели имеют только пользователи из списка `ADMINS` в `config.js`
- Все операции логируются в консоль
- При ошибках пользователь не получает уведомление (только админ видит ошибку)
- ID пользователей валидируются (должны быть числами)
- Количество генераций валидируется (должно быть положительным числом)

## Примеры использования

### Добавить 50 генераций пользователю с ID 123456789

1. `/start` → `👥 Пользователи`
2. Отправить: `123456789`
3. Нажать: `➕ Добавить генерации`
4. Нажать: `+50`
5. Готово! ✅

### Удалить 10 генераций пользователю с ID 987654321

1. `/start` → `👥 Пользователи`
2. Отправить: `987654321`
3. Нажать: `➖ Убрать генерации`
4. Нажать: `-10`
5. Готово! ✅

### Добавить произвольное количество (например, 250)

1. `/start` → `👥 Пользователи`
2. Отправить: `123456789`
3. Нажать: `➕ Добавить генерации`
4. Нажать: `✏️ Ввести вручную`
5. Отправить: `250`
6. Готово! ✅

## Устранение неполадок

**Проблема:** Пользователь не найден
- Проверьте правильность ID (должен быть число)
- Убедитесь, что пользователь запускал бота хотя бы раз

**Проблема:** Уведомление не дошло до пользователя
- Пользователь мог заблокировать бота
- Операция все равно выполнена успешно
- В логах будет предупреждение `Could not notify user`

**Проблема:** Ошибка при добавлении/удалении
- Проверьте подключение к Redis
- Проверьте логи бота в консоли
- Убедитесь, что UserService работает корректно
