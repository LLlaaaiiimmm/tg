## 🐛 Найденные критические баги

### БАГ #1: Регистрозависимая проверка статуса ⚠️⚠️⚠️
**Критичность:** 🔴 КРИТИЧЕСКАЯ

**Проблема:**
```javascript
if (status === 'success' || status === 'paid')  // ❌ НЕ РАБОТАЕТ!
```

0xProcessing отправляет `Status: "Success"` (с большой буквы), поэтому проверка **ВСЕГДА** возвращала false!

**Это означает, что ВСЕ крипто-платежи НЕ ОБРАБАТЫВАЛИСЬ автоматически!**

**Исправление:**
```javascript
const isSuccess = status && (
    status.toLowerCase() === 'success' || 
    status.toLowerCase() === 'paid' || 
    status.toLowerCase() === 'completed'
);
```

---

### БАГ #2: Использование несуществующего clientId
**Критичность:** 🔴 КРИТИЧЕСКАЯ

**Проблема:**
```javascript
const { billingID, status, clientId } = req.body;
await userService.addPaidQuota(clientId, pkg.generations);  // ❌ clientId = undefined
```

По документации 0xProcessing НЕ отправляет поле `clientId`. Отправляются `MerchantId` и `ShopId`.

**Исправление:**
```javascript
await userService.addPaidQuota(order.userId, pkg.generations);  // ✅ Из заказа
```
