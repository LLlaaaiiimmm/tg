# ✅ Исправление парсинга криптовалют для пакетов

## Проблема
При выборе криптовалюты для пакетов pack_10, pack_100, pack_300 система неправильно парсила callback_data:

```
Было:
crypto_USDT_pack_10 → crypto="USDT_pack", packageKey="10" ❌
crypto_USDC_pack_100 → crypto="USDC_pack", packageKey="100" ❌
crypto_TON_pack_300 → crypto="TON_pack", packageKey="300" ❌
```

Из-за этого система не могла найти криптовалюту "USDT_pack" в списке поддерживаемых и возвращала ошибку:
```
❌ No chains found for crypto: USDT_pack
```

## Причина
Регулярное выражение `/crypto_(\w+)_(.+)/` было некорректным:
- `\w+` захватывает ВСЕ буквы, цифры И подчеркивания
- Для "crypto_USDT_pack_10" оно захватывало "USDT_pack" как первую группу

## Решение
Изменили регулярное выражение на `/crypto_([A-Z]+)_(.+)/`:
- `[A-Z]+` захватывает ТОЛЬКО заглавные буквы (USDT, USDC, TON)
- Теперь парсинг работает корректно

```
Стало:
crypto_USDT_pack_10 → crypto="USDT", packageKey="pack_10" ✅
crypto_USDC_pack_100 → crypto="USDC", packageKey="pack_100" ✅
crypto_TON_pack_300 → crypto="TON", packageKey="pack_300" ✅
```

## Исправленный файл
**Файл:** `/app/meemee_bot/src/bot_start.js`

**Строка 504:**
```javascript
// Было:
bot.action(/crypto_(\w+)_(.+)/, (ctx) => {

// Стало:
bot.action(/crypto_([A-Z]+)_(.+)/, (ctx) => {
```
